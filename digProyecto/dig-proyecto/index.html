La Psicología de la Forma

Las leyes de la Gestalt explican cómo el cerebro humano organiza la percepción visual en estructuras coherentes. A continuación, cuatro demostraciones creadas únicamente con HTML y CSS — sin imágenes externas.

Ley de Cierre

El cerebro completa formas incompletas para percibir figuras cerradas.

Ley de Proximidad

Los elementos cercanos entre sí se perciben como un grupo unitario.

Bloque II

Laboratorio de Color

Los espacios de color determinan cómo representamos y manipulamos el color en pantalla. Usa el conmutador de temas de la barra superior para explorar Claro, Oscuro y Alto Contraste.

🔴🟢🔵 Espacio RGB

RGB (Red, Green, Blue) define cada color como una mezcla de tres canales de luz aditiva. Cada canal va de 0 a 255. Es el modelo nativo de las pantallas.

Ejemplo: rgb(220, 53, 69) es un rojo intenso; mezclar los tres al máximo produce blanco rgb(255,255,255).

🌈 Espacio HSL

HSL (Hue, Saturation, Lightness) es más intuitivo: el matiz va de 0° a 360° en la rueda de color, la saturación y la luminosidad del 0 % al 100 %.

Es ideal para ajustar tonos armónicos simplemente rotando el matiz. Ejemplo: hsl(200, 70%, 50%) es un azul vivo.

◑ Contraste y WCAG

Las WCAG 2.1 exigen un ratio de contraste mínimo de 4.5:1 (nivel AA) para texto normal. Textos grandes o de interfaz requieren al menos 3:1.

✓ PASS – ratio 7.2:1 (AA + AAA)
✓ PASS – ratio 4.6:1 (AA)
✗ FAIL – ratio 1.9:1 (insuficiente)

Bloque IV

Compromiso con la Accesibilidad

La accesibilidad no es un extra: es un derecho. Esta web implementa múltiples técnicas para garantizar que cualquier persona, independientemente de su capacidad visual o motriz, pueda navegar y comprender su contenido.

🏗️

HTML Semántico

Usar las etiquetas correctas permite a los lectores de pantalla entender la estructura.

<nav> navegación
<div class="nav">
<main> contenido
<article> unidad
<button> acción
<div onclick>
🎤

ARIA y Roles

Los atributos aria-label, aria-pressed y role enriquecen la semántica de elementos interactivos sin equivalente HTML nativo.

role="progressbar" aria-valuenow="68"

🔲

Contraste WCAG AA

Todos los textos de esta web superan el ratio de contraste 4.5:1 exigido por WCAG 2.1 nivel AA, incluso en modo oscuro y en alto contraste.

✓ Texto normal – 7.2:1
✓ Acento frío – 4.6:1
✗ Ejemplo fallido – 1.9:1
⌨️

Navegación con Teclado

Todo elemento interactivo es alcanzable con Tab y activable con Enter o Espacio. El skip link en la cabecera permite saltar al contenido.

⏸️

Reducción de Movimiento

La media query prefers-reduced-motion: reduce desactiva todas las animaciones CSS para usuarios con vestibular sensibility o epilepsia fotosensible.

@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; }
}
🖼️

Texto Alternativo y Alt

Cada elemento visual de esta página incluye un aria-label descriptivo o un atributo alt. Los elementos puramente decorativos tienen aria-hidden="true" para no contaminar la experiencia auditiva.

Autoevaluación Heurística (Nielsen)

Evaluación propia del producto basada en las 10 Heurísticas de Jakob Nielsen.

H1
Visibilidad del estado El conmutador de temas muestra el botón activo con fondo de acento. La barra sticky confirma la sección en pantalla.
H2
Coincidencia sistema–mundo real Las metáforas (☀ claro, ☾ oscuro, ◑ contraste) son reconocibles. Los botones 90s usan terminología de época.

Control y libertad El usuario puede cambiar de tema en cualquier momento. El skip link permite obviar la nav.

Consistencia y estándares Variables CSS centralizadas garantizan coherencia tipográfica y cromática en los tres temas.
H5
Prevención de errores Los botones tienen estados :hover y :focus claros. Ninguna acción es destructiva.
H6
Reconocer antes que recordar Los iconos y etiquetas en la nav y los botones de tema reducen la carga cognitiva del usuario.
H7
Flexibilidad y eficiencia El skip link acelera la navegación para usuarios expertos. Los anclajes permiten ir directamente a cada bloque.
H8
Diseño estético y minimalista Cada sección tiene un único foco visual. No hay información redundante. Los espacios en blanco actúan como respiraderos.
H9
Ayuda a reconocer errores Los botones no funcionales de demo están etiquetados como "Ejemplo". No se generan estados de error sin retroalimentación.

Ayuda y documentación Cada concepto técnico incluye una descripción breve. Los fragmentos de código están comentados.
